14. QA TESTING — VERIFY EVERY NUMBER
================================================================================

Your app's reputation lives or dies on accuracy. One wrong ampacity
value = 1-star review = ranking death. Test once, test right.

SOURCE OF TRUTH:
  Your physical 2026 code book. Every number traces to a specific
  table and article. No substitute.

TEST CASES PER CALCULATOR (5 each):
  1. Normal case — typical everyday values
  2. Edge case low — smallest wire, shortest run
  3. Edge case high — largest wire, longest run
  4. Boundary case — right at a threshold where answer changes
  5. Known gotcha — the tricky NEC exception that trips people up

  ~8 tabs × 5 cases = 40 test cases. One afternoon with the book.

SHORTCUT FOR TABLE LOOKUPS:
  Photo the NEC table. Photo your app's output. Put them side by
  side. Visually scan for mismatches. Eyes catch differences faster
  than comparing numbers one at a time.
